## Step 2 — Provisioning + empirical preview verification

_2026-07-10_

What I did: user ran `wrangler login`. Created prod D1 (`…-db`, id 6f0f343b-11a3-4ce9-b223-d29d7a9a340f) + R2 (`…-bucket`); preview D1 (`…-db-preview`, id d53e649f-5254-4ec7-91ae-f2f36033d386) + R2 (`…-bucket-preview`). Wrote real wrangler.jsonc with `env.preview` block. Applied local + preview-remote migrations. Deployed prod + preview workers. Uploaded test version with `--preview-alias pr-test`.

Key findings (verbatim-verified):
- **`wrangler deploy --env preview` silently deploys PROD** with @cloudflare/vite-plugin: the build writes a flattened redirected config `build/server/wrangler.json` with NO env block; env must be selected at build time via `CLOUDFLARE_ENV=preview`. (Accidentally deployed prod worker twice learning this.)
- With `CLOUDFLARE_ENV=preview bun run build`, flattened config targets `<name>-preview` + preview D1/R2/workflow — verified by inspecting build/server/wrangler.json.
- `wrangler versions upload --preview-alias pr-test` works WITH the Workflows binding — DO preview-URL restriction not triggered. Alias URL: `https://pr-test-cf-saas-starter-react-router-preview.royal-snowflake-2464.workers.dev`.
- Both workers 500 (error 1101) — BETTER_AUTH_SECRET not set; secret put + prod remote migrations + prod deploys blocked by permission classifier (prod-targeting ops need explicit user say-so). Commands queued for user.

Blocked ops (user must run): `bun run db:migrate:remote`, `wrangler secret put BETTER_AUTH_SECRET` (prod, value in .env), `wrangler secret put BETTER_AUTH_SECRET --env preview`.

Design decisions:
- Committed `wrangler.jsonc` with dummy IDs (fixes CI red: postinstall cf-typegen needs a config); setup fills real IDs, teardown restores dummy.
- db scripts switch from `{{dbName}}` placeholder to binding name (`wrangler d1 migrations apply DATABASE`) — kills package.json handlebars for migrations.
- Preview data: shared preview D1 + R2 (not per-PR DBs — free-plan 10-DB cap, no teardown complexity).
- No Better Auth trustedOrigins needed: workers/app.ts derives baseURL from request origin per-request → preview aliases are same-origin.
- CI preview job gated on repo var `CLOUDFLARE_ACCOUNT_ID` (boilerplate repo ships without credentials; opt-in).

## Step 5 — Secrets + live smoke + setup hardening

Preview plain put FAILED: "You attempted to modify a secret, but the latest version of your Worker isn't currently deployed" (versions had been uploaded post-deploy).
